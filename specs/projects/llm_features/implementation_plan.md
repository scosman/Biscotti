# Implementation Plan: LLM Features

Ordered, dependency-driven phases. Each phase is independently reviewable and must land **green on `lint` + `test` + `build`** (app-touching phases also run `build-app`). Details live in `functional_spec.md`, `ui_design.md`, `architecture.md`, and `components/*`.

> **Manual-test note:** this project does **not** modify `Packages/LocalLLM`, `Packages/Transcription`, or `Packages/AudioCapture`, so the `ac_*`/`tx_*`/`llm_*` manual-test results are **not** invalidated. (Phase 12 added a *new* manual-test script for the app-level AI features.) Phases 7–11 touch the `Intelligence` module and AI-related UI, so they **invalidate the `ai_*` manual-test results** — each must mark the affected `ai_*` steps `not-run` (per the repo CLAUDE.md staleness rule); the human re-runs them in Phase 12.

> **Polish round (Phases 7–11):** added after Phases 1–6 shipped and the UI was reviewed on-screen. They implement `functional_spec.md` §13 / `ui_design.md` §7. The human end-to-end verification phase (originally Phase 7) is **renumbered to Phase 12** and moved to the end so it validates the polished build in one pass.

## Phases

- [x] **Phase 1 — Data model & DataStore.** Add `Meeting.summary`/`editedSummary`, `TranscriptRecord.speakerAssignments` (JSON `[Int: UUID]`), `AppSettings.summarizeTranscripts`/`guessSpeakerNames`. Extend DTOs (`SegmentData.speakerID`, `TranscriptData.speakerAssignments` resolved to `PersonData`, `MeetingDetailData.summary`/`editedSummary`, `AppSettingsData`). Add `DataStore` methods (`applyGeneratedSummary`, `setSummary`, `setSpeakerAssignments`, `setSpeakerAssignment`, `allPersonData`) and resolve new fields in `meetingDetail`/`transcript`. Additive schema (no V2). **Tests:** DataStoreTests for every field/method, `[Int:UUID]` round-trip, read-model resolution, dangling-ID drop. *(No LLM/UI yet.)*

- [x] **Phase 2 — `Intelligence` module core.** Add `LocalLLM` dep to BiscottiKit; create the `Intelligence` target + product + test target. Build `LLMRunning`/`LLMSession`/`ModelProviding` protocols + live impls, `IntelligencePrompts` (Swift-constant catalog), `TranscriptFormatter`, `SpeakerMappingParser`, `SpeakerIdentifier`, `Summarizer`, the `Intelligence` service (orchestration, `runAutoEnhancements`, `generateSummary`, download state), and the status/download enums. **Tests:** full IntelligenceTests with fakes — gating, ordering (speakers→summary), edited-summary guard, single-session, streaming accumulation, parser cases, download state machine, cancellation. *(Compiles + tests standalone; no app wiring.)*

- [x] **Phase 3 — AppCore wiring & auto-run.** `AppCore` owns a live `Intelligence`; `stopRecording()` triggers `runAutoEnhancements` after `transcribe()`. App target embeds `llama.framework`; verify a hosted connection launches from the app process (`build-app` + smoke). **Tests:** AppCoreTests (trigger-after-transcription, skip when off/no-model, via fakes). *(Milestone: app now links/loads LocalLLM.)*

- [x] **Phase 4 — Settings: AI Enhancements + model download.** `SettingsView` section (two toggles + conditional download row), `SettingsViewModel` additions, download-row state machine bound to `Intelligence.download`/`downloadModel()`, no-model disabling, live flip on completion. **Tests:** SettingsUITests (toggle persistence + revert, no-model state, download state machine). *(Lets a human download the model + toggle features for later manual testing.)*

- [x] **Phase 5 — Meeting detail: Summary tab.** New first `Tab.summary`; `summaryTabContent` state machine (streaming / editable / empty×{no-model, off, ready} / error / no-transcript); VM summary state + debounced edit→`setSummary`; "Regenerate Summary" overflow item + edited-summary confirm; tab-bar status pill; Generate/Regenerate via `Intelligence.generateSummary`; observe enhancement completion → reload. **Tests:** MeetingDetailUITests (tab states, regenerate gating, streaming display, pill visibility, copy).

- [x] **Phase 6 — Meeting detail: Speaker names + mapping sheet.** `TranscriptContent` name param (color keyed by `speakerID`, cache-key includes names); `SpeakerLink` clickable speaker spans; `SpeakerMappingSheet` (invitees → people → add-by-name → unassigned, apply-on-change); VM sheet assembly via `allPersonData`/calendar + `setSpeakerAssignment`. **Tests:** name-replacement rendering, color stability, sheet option assembly + assign/clear; works model-free.

- [x] **Phase 7 — Processing-pipeline status control (Summary tab) + remove pill + auto-jump + re-transcribe.** *(Impl: `functional_spec.md` §13.1; `ui_design.md` §7.1.)* Build a combined pipeline status in `MeetingDetailViewModel` that merges `TranscriptionService.jobs[id]` (`.downloadingModel`/`.transcribing`) and `Intelligence.jobs[id]` (`.identifyingSpeakers`/`.summarizing`) into an ordered, gated stage model (`Transcribing → Inferring participant names → Summarizing`; AI stages included only when the toggle is on **and** a model is present; summary also gated on `!editedSummary`). New `EnhancementPipelineView` (MeetingDetailUI) renders done/active/pending rows using the existing small-spinner idiom; it **replaces** the Summary tab's `summaryNoTranscriptContent` branch whenever the pipeline is active. **Remove** `enhancementPill` and its `isEnhancing` rendering in `tabBar`. **Auto-jump** to `.summary` once when the pipeline becomes active for the open meeting (don't fight later manual tab changes). **Wire `MeetingDetailViewModel.reTranscribe` to call `intelligence.runAutoEnhancements` after `reTranscribe`** (closes the §5.1 gap). **Tests:** combined-status → stage-model mapping; stage gating by settings/model/`editedSummary`; pipeline replaces no-transcript placeholder; pill removed (`isEnhancing` no longer rendered); jump-once-on-active; re-transcribe triggers auto-run. **Marks `ai_*` not-run.**

- [x] **Phase 8 — Fix summary streaming→final flash & scroll loss.** *(Impl: `functional_spec.md` §13.2; `ui_design.md` §7.2.)* Close the empty-content window at stream end: ensure `summaryText` holds the final markdown at the instant `streamingSummary` clears (reorder the completion/reload so the view never falls through to the empty/Generate/`summarySettingsHintContent` state — e.g. seed `summaryText` from the final streamed value, or have `Intelligence` retain `streamingSummary` until the VM's `load()` repopulates `summaryText`). Render streaming and final summary through **one `MarkdownEditor`** instance — drop the `"<id>-summary-streaming"` `documentId`, use the single `"<id>-summary"` id and flip `isEditable` `false`→`true` — so the editor isn't recreated and scroll is retained. **Tests:** completion path never transits the empty/Generate state when a non-empty summary was produced; streaming and final use the same `documentId` (editor identity stable). **Marks `ai_*` not-run.**

- [ ] **Phase 9 — Settings polish: header caption + section order.** *(Impl: `functional_spec.md` §13.3; `ui_design.md` §7.3.)* Move "AI runs locally on your Mac." from the `aiEnhancementsSection` Section `footer` to **muted text trailing the section header** (header row: `Text("AI Enhancements")` · `Spacer()` · `inkSecondary` caption in `metadataFont`). Reorder `SettingsView`'s `Form` so **Permissions is 2nd, after General**: General → Permissions → AI Enhancements → Notifications → Calendars (Debug last). **Tests:** section order assertion (and header-caption presence) as far as the existing SettingsUITests structure allows. *(Non-AI UI; does not touch Intelligence — no `ai_*` invalidation, but harmless to keep in the same human pass.)*

- [ ] **Phase 10 — Speaker-assignment provenance (`userSet`) — never overwrite manual assignments.** *(Impl: `functional_spec.md` §13.4.)* Change the per-transcript speaker map stored value from a bare `UUID` to a small `Codable` struct `{ personID: UUID, userSet: Bool }` on `TranscriptRecord` (JSON-backed via the same `Data` + `@Transient` pattern; **lenient decode** of the old `[Int: UUID]` shape → empty). Manual write paths (`DataStore.setSpeakerAssignment` from the sheet's assign / add-person / re-assign) set `userSet = true`. The LLM bulk write (`setSpeakerAssignments` via `SpeakerIdentifier`) **merges, skipping any speaker whose current entry is `userSet = true`**, and writes AI entries as `userSet = false`. Resolve provenance through the read model only as needed (display still uses `[Int: PersonData]`; person-id for color comes in Phase 11). **Tests:** auto-run preserves a `userSet` entry and fills only unset speakers; manual set marks `userSet`; old-shape decode → empty; unassign clears. **Marks `ai_*` not-run** (Intelligence write path changes).

- [ ] **Phase 11 — Shared color for merged speakers.** *(Impl: `functional_spec.md` §13.5; `ui_design.md` §7.4.)* Re-key transcript speaker color on the **assigned `Person.id`** when a speaker is assigned (so multiple speaker IDs → one person → one color), falling back to the existing `"speaker-<id>"` key when unassigned. Plumb a per-speaker `Person.id` (or precomputed color key) map — derived from `displayedTranscript.speakerAssignments: [Int: PersonData]` — into `TranscriptContent.speakerColor`/`attributedString`, and add it to `CachedTranscriptKey` so the render cache invalidates on assignment changes. Apply the same rule to the `SpeakerMappingSheet` row color dots. **Tests:** two speaker IDs → same person → identical color; unassigned speakers keep their stable per-ID color; cache key includes the person-color map. *(Display-layer only; no Intelligence change — but keep in the Phase 12 human pass.)*
