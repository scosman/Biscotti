# Onboarding Redesign

Visually refresh the first-run onboarding flow to the **F · Sage + Pressroom**
design identity already used by Home and Meeting Detail — **without changing
behavior**. This is a visual refresh, not a functional rework.

## Goal

Take the spirit of the design brief below (generated by a design agent) and
realize it in native SwiftUI/macOS using Biscotti's **existing** `DesignSystem`
tokens and conventions. The brief is good at look-and-feel but is "web-like" in
places and does not know our latest coding standards or component library —
where it conflicts with the codebase, the **codebase wins**, and we ask when in
doubt.

## Hard constraints

- **Behavior must not change.** Every existing step, permission request, error
  case, denial-guidance path, validation/retry flow, calendar selection,
  launch-at-login, disk-space check, and download-failure handling is preserved.
  This is a reskin of the existing `OnboardingUI` flow, not a redesign of it.
- **Respect the project design system.** Use the existing `DesignSystem`
  tokens/fonts/colors, not the brief's literal names:
  - Serif → `Font.biscottiSerif` (**Newsreader Display**), *not* "Newsreader".
  - Mono → `Font.biscottiMono` (**JetBrains Mono**), *not* "IBM Plex Mono".
  - The brief's `Pal.*` palette maps to existing `Color` tokens
    (`.paper`/`.ink`/`.sage`/`.inkSecondary`/`.inkTertiary`/`.hairline`/
    `.cardStroke`/`.accentWashSoft`/…). No new palette is introduced.
  - Prefer existing shared components/button styles over re-deriving the
    brief's hand-rolled CSS-style gradients/shadows.
- **Light appearance only** (matches the rest of the app today).
- **Don't over-index on the brief's specifics.** Treat its pixel values,
  font names, and `enum`/`struct` code as illustrative intent, not literal
  requirements. Translate to idiomatic SwiftUI for this codebase.

## What "the spirit" means here

The brief's valuable ideas to carry over: a calm, centered, roomy layout; a
single thin sage progress bar with a mono step kicker (replacing the current
dots); a serif step title as the one display moment per screen; a quiet brand
footer lockup; consolidated/cleaner permission presentation; and consistent
sage primary buttons matching Home.

Open structural questions (window model, how far to consolidate steps) are
resolved with the user during the functional spec — see that document.

<details>
<summary>Biscotti Onboarding — SwiftUI Implementation Spec (design agent)</summary>

The brief proposes a standalone centered 960×680 fixed window, light
appearance, F · Sage + Pressroom identity (ivory paper, sage accent,
Newsreader serif title, IBM Plex Mono kicker/numerals). It collapses the flow
to **4 steps** — Welcome, consolidated "Grant access" (one card, one row per
permission, each firing its own native prompt), Download Local AI Models (with
idle/downloading/done states and "Skip for now"), and "You're all set" — with
a thin 240×3 sage progress bar + mono kicker as the only progress affordance,
a sage primary button, a `lock.shield.fill` + "Biscotti" footer lockup, and
cross-fade + small-offset step transitions (reduced-motion aware). Full text
of the brief is retained in the project thread / functional spec discussion.
