# Phase 6: `biscotti://meeting` deep link end-to-end

## Overview

Register the `biscotti` URL scheme and implement the full deep-link pipeline:
the app handles `biscotti://meeting/{id}?time={seconds}` URLs by navigating to
the meeting, switching to the Transcript tab, and seeking the audio playhead to
the requested timestamp. This enables the timestamp links generated by Phase 2's
`NotesMarkdown` to work when clicked in the Notes tab's `MarkdownEditor`.

## Steps

1. **Register `biscotti` URL scheme in `App/project.yml`.**
   Add `CFBundleURLTypes` to the Biscotti target's Info.plist settings so
   XcodeGen generates the scheme registration. Run `generate` + `build_app`.

2. **Add `TranscriptJump` and `pendingTranscriptJump` to `AppCore`.**
   ```swift
   public struct TranscriptJump: Sendable, Equatable {
       public let meetingID: UUID
       public let time: TimeInterval
   }
   public private(set) var pendingTranscriptJump: TranscriptJump?
   public func consumeTranscriptJump() { pendingTranscriptJump = nil }
   ```

3. **Add `handleDeepLink(_:)` to `AppCore`.**
   Parse `biscotti://meeting/{uuid}?time={seconds}` — validate scheme, host,
   UUID, and that the meeting exists. On success: `select(meetingID)` and set
   `pendingTranscriptJump`. Invalid/missing → no-op.

4. **Wire `BiscottiApp` / `AppDelegate` for `onOpenURL`.**
   `WindowRootView` adds `.onOpenURL { appDelegate.handleOpenURL($0) }`.
   `AppDelegate.handleOpenURL` calls `showMainWindow()` then
   `core?.handleDeepLink(url)`.

5. **Lift `Tab` from `MeetingDetailView` into `MeetingDetailViewModel`.**
   Move `enum Tab` to the VM as a public type; add `var selectedTab: Tab = .transcript`.
   Replace the view's `@State private var tab` with `$viewModel.selectedTab`.

6. **Add pending-jump application to `MeetingDetailViewModel`.**
   ```swift
   private var pendingSeek: TimeInterval?
   func applyPendingJumpIfNeeded()  // reads core.pendingTranscriptJump
   func applySeekIfReady()          // seek(to: min(time, duration))
   ```
   Called via `.onChange(of: core.pendingTranscriptJump)` and on appear.
   If audio isn't loaded yet, `pendingSeek` is retained and applied at the
   end of `loadAudioPlayer()`.

7. **Unit tests for URL parsing (AppCore).**
   - Valid `biscotti://meeting/{uuid}?time=42.0` → sets selection + pending jump.
   - Missing `time` query → no-op.
   - Invalid UUID → no-op.
   - Wrong scheme → no-op.
   - Wrong host → no-op.
   - Non-existent meeting ID → no-op.
   - `consumeTranscriptJump` clears.

8. **Unit tests for jump application (MeetingDetailViewModel).**
   - Pending jump for this meeting → `selectedTab == .transcript` + seek
     clamped to duration.
   - Pending jump for a different meeting → ignored.
   - Jump arriving before audio loads → deferred, applied after
     `loadAudioPlayer()`.
   - `consumeTranscriptJump` called after application.

## Tests

- `DeepLinkParsingTests`: valid URL, missing time, bad UUID, wrong scheme,
  wrong host, non-existent meeting, integer time, consume clears.
- `DeepLinkJumpTests`: tab switch + seek, clamp to duration, deferred seek
  until audio loads, ignore mismatched meeting, consume after apply.
