# Phase 4.4: CI Gate + CLAUDE.md Convention + Seed Results File

## Overview

Wire the manual-test results file into CI so that unrun tests block merges, and add a CLAUDE.md convention that resets manual tests when their underlying libraries change. This makes the manual-test contract enforceable: agents that modify Transcription or AudioCapture automatically trigger re-runs.

The gate is **deliberately red** after this phase (all steps are `not-run`). Phase 4.5 turns it green when a human runs the tests.

## Steps

### 1. Seed results file

Create `ManualTestApp/Results/manual_test_results.json` containing every step ID from both canonical scripts (`allScripts`), each with `status: "not-run"`. The file is generated by enumerating `TestScript.audioCapture` and `TestScript.transcription` step IDs. Checked in.

Step IDs (21 total):
- Audio Capture (12): `ac_request_permissions`, `ac_two_dialogs`, `ac_timed_capture`, `ac_start_recording`, `ac_stop_recording`, `ac_files_exist`, `ac_playback_mic`, `ac_playback_system`, `ac_route_change`, `ac_crash_safety_setup`, `ac_crash_safety_check`, `ac_monitoring`
- Transcription (9): `tx_model_download`, `tx_model_disk`, `tx_transcribe`, `tx_speakers`, `tx_no_hallucination`, `tx_crash_setup`, `tx_crash_host_survives`, `tx_crash_retry`, `tx_custom_vocab`

### 2. Results path reconciliation

Phase 4.2's app writes to `~/Documents/ManualTestApp/results.json`. Change `ScriptTabView.resultsFileURL()` to resolve the repo-relative `ManualTestApp/Results/manual_test_results.json` instead. Strategy: use the `MANUAL_TEST_RESULTS_PATH` environment variable if set, otherwise locate the repo root by walking up from `Bundle.main.bundlePath` to find `ManualTestApp/Results/`. Fallback: document that the human should set the env var or copy the file. The simplest reliable approach: detect that we're inside the repo by looking for the `ManualTestApp/Results` directory relative to the source root (Xcode sets `SOURCE_ROOT` via the build settings), and fall back to `~/Documents` if not found.

Actual implementation: use `MANUAL_TEST_RESULTS_PATH` env var if set; else use the Xcode `SOURCE_ROOT` build setting (exposed via `SRCROOT` in the process env by xcodebuild/Xcode) to find `../ManualTestApp/Results/manual_test_results.json`; else fall back to `~/Documents/ManualTestApp/manual_test_results.json`. This ensures the app writes directly to the repo-tracked file during development.

### 3. SPM executable target `manual-tests-check`

Add an executable target to `Packages/BiscottiKit/Package.swift`:
- Name: `manual-tests-check`
- Source: `Sources/manual-tests-check/main.swift`
- Dependencies: `ManualTestKit`
- Logic: parse a single CLI argument (results file path), create `ResultsStore(fileURL:)`, call `unrun(in: allScripts)`, print the unrun IDs, exit 1 if non-empty, exit 0 if all run.

### 4. Makefile target

Add `manual-tests-check` to the Makefile:
```
manual-tests-check: ## Check that all manual test IDs have been run
    swift run --package-path Packages/BiscottiKit manual-tests-check ManualTestApp/Results/manual_test_results.json
```
Add to `.PHONY`. Do NOT add to `ci` target (it must stay `lint test build`).

### 5. CI job

Add a `manual-tests-check` job to `.github/workflows/ci.yml`:
- Separate job, NOT part of `package-tier` or `app-tier`
- `continue-on-error: true` so the overall CI status isn't blocked (the gate is informational like app-tier until Phase 4.5)
- Comment noting it's expected-red until manual tests are run
- Minimal setup: just checkout + `make manual-tests-check`

### 6. hooks_mcp.yaml action

Add a `manual_tests_check` action wrapping `make manual-tests-check` with bounded output (same tail pattern as other compile actions).

### 7. CLAUDE.md convention

Add a section to the root CLAUDE.md documenting the staleness rule: when you touch `Packages/Transcription` or `Packages/AudioCapture`, mark that library's manual tests `not-run` by hand-editing the results file or using `ResultsStore.markScriptNotRun`.

### 8. Gate-logic unit test against seed file

Add a test to `CIGateTests.swift` that loads the actual seed file (`ManualTestApp/Results/manual_test_results.json`) and confirms `unrun(in: allScripts)` returns all 21 step IDs (proving the seed file is correctly shaped and the gate logic correctly identifies it as not-all-run).

## Tests

- `seedFileReportsAllUnrun` (in CIGateTests): loads the real seed file, verifies all 21 step IDs are reported as unrun, and that the set matches exactly `allStepIDs(in: allScripts)`.
